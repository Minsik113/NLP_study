● 텍스트 유사도(Text Similarity)
: 문장이 유사한지 측정
ex) 1. 인공지능 스피커에게 이 노래 누가만들었어?
    2. 지금 들리는 노래의 작곡가가 누구야?
    -> 이 둘은 같은 의미를 갖지만, 컴퓨터에게도 같은 문장은 아니다. 이를 어떻게 이해시킬 것인가?

유사도를 판단하는 척도는 매우 주관적이기 때문에 데이터를 구성하기가 쉽지 않고 정량화하는 데 한계가 있다.
-> 최대한 정량화해서 모델을 만드는 것이 중요하다.
- 유사도를 측정하기 위해 정량화하는 방법:
    단순히 단어의 개수를 사용, 형태소로 나누어 형태소비교, 자소 단위로 나누어 단어를 비교 등등

=> 우리는 텍스트를 벡터화한 후 벡터화된 각 문장간의 유사도를 측정하는 방식을 배울것이다.

- 종류:
    자카드 유사도, 유클리디언 유사도, 맨해튼 유사도, 코사인 유사도
    1. TF-IDF(Term Frequency - Inverse Document Frequency)로 벡터화한 값은 자카드 유사도를 제외한 유사도 측정에 모두 사용할 것이다.
    (TF-IDF는 '사이킷런'에서 나옴)

● 자카드 유사도(Jaccard Similarity) = 자카드 지수
: 두 문장을 각각 단어의 집합으로 만든 뒤 두 집합을 통해 유사도를 측정하는 방식 중 하나이다.
- 방법:
    교집합 / 전체집합 => 1에 가까울 수록 유사도가 높다.(당연)

● 코사인 유사도 (유사도 계산시 가장 널리 쓰이는 방법 중 하나 - 성능이 좋으니까 - 각도를 구하는 것이므로 방향성의 개념이 더해지니까)
: 두 개의 벡터값에서 코사인 각도를 구하는 방법이다. -1 ~ 1 사이의 값을 가지며 1에 가까울 수록 유사하다.

● 유클리디언 유사도
: 가장 기본적인 거리를 측정하여 유사도를 구하는 방식이다.
(우리가 아는 점사이의 거리공식)
- 방법:
    1. a = 점 사이의 거리값 
    2. 정규화로 0~1사이의 값을 갖게 한다. (L1 정규화 방법 사용)
    (L1 정규화 방법 = 각 벡터 안의 요소 값을 모두 더한 것이 크기가 1이 되도록 벡터들의 크기를 조절하는 방법이다.
    즉, 벡터의 모든 값을 더한 후 이 값(모두 더한 값)으로 각 벡터의 값을 나누면 된다.)

● 맨해튼 유사도(Manhattan Similarity)
: 맨해튼 거리를 통해 유사도를 측정하는 방법이다.
맨해튼 거리란, 사각형 격자로 이뤄진 지도에서 출발점에서 도착점까지를 가로지르지 않고 갈 수 있는 최단거리를 구하는 공식이다.
    유클리디언 거리 = L2-Distance
    맨해튼 거리     = L1-Distance 라 부름
- 방법:
    1. 두 점사이의 맨해튼 거리를 구함
    2. 정규화한다. 0~1의 값을 갖게 한다.
